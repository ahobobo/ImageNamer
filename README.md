# ImageNamer

ImageNamer is a .NET console app that renames image files with descriptive names generated by a local Ollama model.

You give it either a single image file or a directory. If you pass a directory, it searches recursively for supported image files, asks the model for a better filename, and renames each image in place.

## What It Does

- Accepts a single path argument: either a file path or a directory path.
- Supports these image extensions:
  - `.jpg`
  - `.jpeg`
  - `.png`
  - `.gif`
  - `.bmp`
  - `.webp`
- Processes directories recursively.
- Skips non-image files.
- Renames files in the same folder as the original image.
- Lowers the final filename before saving it.
- Continues processing even if one file fails.

The model is instructed to produce a short descriptive filename, usually under 20 characters, using only letters, numbers, and spaces.

## Requirements

Before running the app, install and configure:

- [.NET 10 SDK](https://dotnet.microsoft.com/download)
- [Ollama](https://ollama.com/)
- The Ollama model used by the app: `gemma4:e4b`

The app connects to Ollama at `http://localhost:11434`, so Ollama must be running locally when you launch the CLI.

If you have not already pulled the model, run:

```powershell
ollama pull gemma4:e4b
```

## How To Run

From the repository root, run the console project and pass the file or directory path after `--`:

```powershell
dotnet run --project .\Console\Console.csproj -- "C:\path\to\image-or-folder"
```

### Usage

```powershell
dotnet run --project .\Console\Console.csproj -- <file_path>
dotnet run --project .\Console\Console.csproj -- <directory_path>
dotnet run --project .\Console\Console.csproj -- --help
dotnet run --project .\Console\Console.csproj -- -h
dotnet run --project .\Console\Console.csproj -- /?
```

Examples:

```powershell
dotnet run --project .\Console\Console.csproj -- .\TestData
```

```powershell
dotnet run --project .\Console\Console.csproj -- "C:\Photos\Vacation"
```

```powershell
dotnet run --project .\Console\Console.csproj -- "C:\Photos\Vacation\IMG_0012.webp"
```

## Notes

- If the path does not exist, the app exits with an error.
- If a directory contains no supported images, it reports that nothing was found.
- If any rename fails, the app keeps going and exits with a non-zero exit code.
- The command line does not take flags or options, only the input path.

## Project Layout

- `Console/` - CLI entrypoint and Ollama wiring
- `Application/` - image renaming use case and ports
- `Infrastructure/` - file handling and Ollama integration
- `ApplicationTests/` - automated tests
